# [시스템 기획] 무기_반동

생성자: YUCHAN BAE  
카테고리: 기획 / 구현 설계  
최종 갱신: 2026년 4월 28일

> **작성 목적:** 무기별 반동 데이터를 C++ 데이터 구조와 카메라 피드백 시스템에 연결하고, TPS 조준 카메라 구도를 유지하는 반동 적용 원칙을 명세한다.

---

## 1. 개요

무기 반동은 `UPRWeaponDataAsset`의 `FPRRecoilProfile` 데이터로 관리한다. 사격 어빌리티는 발사 성공 시 현재 활성 무기의 반동 프로파일을 이벤트로 전달하고, 로컬 클라이언트의 `UPRCameraModifier_Recoil`이 해당 이벤트를 받아 카메라 회전 반동과 카메라 셰이크를 처리한다.

반동은 서버 판정 데이터가 아니라 로컬 시각 피드백이다. 서버는 기존 사격 동기화 규칙에 따라 발사 방향, 발사 위치, 피격 결과만 검증한다. 카메라 반동, 카메라 셰이크, 크로스헤어 확산은 서버로 복제하지 않는다.

## 3. 데이터 구조

### 3.1 FPRRecoilProfile

`FPRRecoilProfile`은 무기별 반동, 크로스헤어 확산, 카메라 셰이크 데이터를 담는다.

| 필드 | 설명 |
|---|---|
| `RecoilCurve` | 연속 발사 횟수 기준 반동 커브. `X = Pitch`, `Y = Yaw`, `Z = 예비값` |
| `FallbackPitch` | 커브가 없을 때 사용할 기본 수직 반동 |
| `FallbackYawMin` | 커브가 없을 때 사용할 최소 수평 반동 |
| `FallbackYawMax` | 커브가 없을 때 사용할 최대 수평 반동 |
| `RecoilSpeed` | 현재 카메라 반동값이 목표 반동값으로 따라가는 속도 |
| `RecoilRecoverySpeed` | 사격 중단 후 반동이 원상 복구되는 속도 |
| `RecoveryDelay` | 마지막 발사 후 복구 시작까지의 지연 시간 |
| `ADSRecoilMultiplier` | ADS 상태에서 회전 반동에 곱할 배율 |
| `CameraRecoilScale` | 커브 또는 fallback 반동값을 실제 카메라 회전값으로 줄이는 배율 |
| `MaxRecoilPitch` | 카메라 Pitch 반동 최대 누적값 |
| `MaxRecoilYaw` | 카메라 Yaw 반동 최대 누적값 |
| `CrosshairSpreadIncrease` | 발사 1회당 크로스헤어 확산 증가량 |
| `MaxCrosshairSpread` | 크로스헤어 확산 최대값 |
| `CrosshairRecoverySpeed` | 크로스헤어 확산 회복 속도 |
| `CameraShakeClass` | 사격 반동과 함께 재생할 카메라 셰이크 클래스 |
| `CameraShakeScale` | 카메라 셰이크 재생 강도 |
| `ADSCameraShakeMultiplier` | ADS 상태에서 카메라 셰이크에 곱할 배율 |

### 3.2 커브 적용 규칙

`RecoilCurve`가 지정되어 있으면 연속 발사 횟수를 커브 입력값으로 사용한다.

```text
CurveTime = ConsecutiveShots
Pitch = CurveVector.X
Yaw = CurveVector.Y
```

`RecoilCurve`가 없으면 fallback 값을 사용한다.

```text
Pitch = FallbackPitch
Yaw = Random(FallbackYawMin, FallbackYawMax)
```

이후 ADS 배율과 카메라 반동 배율을 적용한다.

```text
FinalRotationRecoil = RawRecoil * ADSRecoilMultiplier * CameraRecoilScale
```

ADS가 아닌 상태에서는 `ADSRecoilMultiplier`를 적용하지 않는다.

## 4. TPS 카메라 반동 원칙

### 4.1 FPS식 무한 누적 금지

TPS에서는 카메라 회전을 무한 누적하면 플레이어 캐릭터가 화면 아래로 밀려 사라진다. 따라서 반동은 조준 카메라 축 기준의 제한된 오프셋으로 처리한다.

금지 방식:

```cpp
TargetRecoilRot += AddRecoil;
```

적용 방식:

```cpp
TargetRecoilRot.Pitch = FMath::Clamp(
	TargetRecoilRot.Pitch + AddRecoil.Pitch,
	-Profile.MaxRecoilPitch,
	Profile.MaxRecoilPitch);

TargetRecoilRot.Yaw = FMath::Clamp(
	TargetRecoilRot.Yaw + AddRecoil.Yaw,
	-Profile.MaxRecoilYaw,
	Profile.MaxRecoilYaw);
```

### 4.2 PRGA_PlayerAim 기준 카메라 축 유지

반동은 `PRGA_PlayerAim` 상태에서 만들어진 OTS 조준 카메라 구도를 깨지 않는 범위에서만 적용한다. 실제 명중 불안정성의 큰 부분은 카메라 회전 누적이 아니라 크로스헤어 확산, 사격 방향 변화, 카메라 셰이크로 분산한다.

권장 기본값:

| 값 | 권장 범위 |
|---|---|
| `CameraRecoilScale` | `0.2 ~ 0.35` |
| `MaxRecoilPitch` | `1.5 ~ 3.0` |
| `MaxRecoilYaw` | `0.5 ~ 1.5` |

돌격소총은 `MaxRecoilPitch`를 낮게 잡고, 볼트액션과 유탄발사기는 단발 임팩트를 위해 `CameraRecoilScale`과 카메라 셰이크를 상대적으로 강하게 둔다.

## 5. 카메라 셰이크

### 5.1 처리 주체

카메라 셰이크는 `UPRCameraModifier_Recoil`에서 처리한다. `UPRGA_Fire`는 셰이크를 직접 실행하지 않고 `Event.Player.Recoil`만 발송한다.

```text
FireOneShot
-> Event.Player.Recoil
-> UPRCameraModifier_Recoil
-> 회전 반동 적용
-> CameraShake 재생
```

### 5.2 CameraShakeClass 지정

`CameraShakeClass`에는 `UCameraShakeBase` 계열 Blueprint 클래스를 지정한다.

권장 에셋 예시:

```text
CS_Recoil_AssaultRifle
CS_Recoil_BoltAction
CS_Recoil_GrenadeLauncher
```

`CameraShakeScale`은 보통 `0.1 ~ 2.0` 범위에서 사용한다. `1000`처럼 큰 값은 카메라 위치와 회전 진폭 전체를 과도하게 키우므로 TPS 화면에서 플레이어가 흔들리는 것처럼 보인다.

### 5.3 셰이크 조정 원칙

셰이크가 약하면 `CameraShakeScale`을 과도하게 키우지 말고 CameraShake 에셋 내부의 Amplitude, Frequency, Duration을 조정한다.

TPS에서는 Location Shake가 캐릭터와 카메라 사이 프레이밍을 크게 흔들 수 있으므로, 기본적으로 Rotation Shake를 우선 사용하고 Location Shake는 유탄발사기 같은 강한 충격 무기에만 제한적으로 사용한다.

권장 시작값:

| 무기 | Duration | Pitch Amp | Yaw Amp | Roll Amp | Location |
|---|---:|---:|---:|---:|---|
| 돌격소총 | `0.06 ~ 0.10` | `0.8 ~ 1.2` | `0.2 ~ 0.4` | `0.05 ~ 0.1` | 없음 또는 매우 약함 |
| 볼트액션 | `0.12 ~ 0.20` | `1.5 ~ 2.5` | `0.3 ~ 0.5` | `0.1 ~ 0.2` | 약함 |
| 유탄발사기 | `0.18 ~ 0.30` | `1.2 ~ 2.0` | `0.4 ~ 0.7` | `0.1 ~ 0.25` | Z `1.0 ~ 2.0` |

## 6. 크로스헤어 확산

크로스헤어 확산은 `UPRCrosshairWidget` 또는 해당 BP 위젯에서 독립적으로 계산한다. `Event.Player.Recoil` 수신 시 `FPRRecoilEventPayload`의 `Speed`, `Strength`, `RecoilProfile` 값을 참조할 수 있다.

기획 규칙:

- 발사 시 `CrosshairSpreadIncrease`만큼 확산
- 확산값은 `MaxCrosshairSpread`를 넘지 않음
- 발사 이벤트가 일정 시간 없으면 `CrosshairRecoverySpeed` 기준으로 회복
- ADS 상태에서는 회복 속도 1.5배 적용

현재 C++ 이벤트 페이로드는 기존 BP 호환을 위해 `Speed`, `Strength` 필드를 유지한다.

## 7. 네트워킹 원칙

반동 시스템은 로컬 클라이언트 전용 시각 피드백이다.

| 항목 | 처리 주체 | 복제 여부 |
|---|---|---|
| 카메라 회전 반동 | 로컬 클라이언트 | 복제하지 않음 |
| 카메라 셰이크 | 로컬 클라이언트 | 복제하지 않음 |
| 크로스헤어 확산 | 로컬 HUD | 복제하지 않음 |
| 발사 위치와 방향 | 클라이언트 보고, 서버 검증 | RPC |
| 피해 계산 | 서버 | 서버 권위 |

Listen Server 환경에서도 시각 피드백 직전에는 `IsLocalController()` 또는 `IsLocallyControlled()` 검사를 유지한다. 원격 플레이어의 카메라 반동이나 셰이크를 로컬 화면에서 실행하지 않는다.

## 8. DA_TestRifle 테스트 절차

`DA_TestRifle`을 테스트하려면 플레이어 시작 시 `UPRWeaponManagerComponent::EquipTestWeaponToSlot()`을 호출해 주무기 슬롯에 장착해야 한다.

테스트용 흐름:

```cpp
if (HasAuthority() && IsValid(TestInitialWeaponData))
{
	if (UPRWeaponManagerComponent* WeaponManagerComponent = GetWeaponManager())
	{
		WeaponManagerComponent->EquipTestWeaponToSlot(TestInitialWeaponData, EPRWeaponSlotType::Primary);
	}
}
```

이 방식은 임시 테스트용이다. 정식 장비 시스템이 연결되면 `DefaultLoadout` 또는 장비/인벤토리 플로우에서 무기 데이터가 설정되어야 한다.

## 9. 검증 항목

- `DA_TestRifle`의 `RecoilProfile`에 커브가 없을 때 fallback 반동이 적용되는지 확인
- `RecoilCurve` 지정 시 연속 발사 횟수에 따라 Pitch/Yaw가 달라지는지 확인
- 장시간 풀오토 사격 시 플레이어 캐릭터가 화면 아래로 사라지지 않는지 확인
- `MaxRecoilPitch`, `MaxRecoilYaw` 제한이 적용되는지 확인
- ADS 상태에서 `ADSRecoilMultiplier`, `ADSCameraShakeMultiplier`가 적용되는지 확인
- `CameraShakeScale`을 `0.1 ~ 2.0` 범위에서 조정했을 때 자연스럽게 보이는지 확인
- `CameraShakeScale`을 과도하게 키우지 않고 Shake BP 내부 값을 통해 감도를 조정하는지 확인
- 클라이언트 반동과 셰이크가 서버 또는 다른 클라이언트 화면에 복제되지 않는지 확인

## 10. 남은 작업

- 크로스헤어 BP에서 `FPRRecoilEventPayload` 기반 확산 회복 로직 구체화
- 무기별 `UCurveVector` 에셋 작성
- `DA_TestRifle`을 포함한 테스트 무기 데이터에 `RecoilProfile` 입력
- 돌격소총, 볼트액션, 유탄발사기 전용 `CameraShakeClass` 제작
- 정식 장비 시스템 연결 후 테스트용 장착 코드 제거
