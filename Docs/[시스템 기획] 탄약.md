## 1. 시스템 목표

탄약 시스템은 플레이어가 같은 탄약 픽업으로도 어떤 무기를 들고 있느냐에 따라 **체감 발사 횟수가 달라지는 경험**을 제공한다. 동시에 무기를 교체했을 때 모은 탄약이 자연스럽게 새 무기 기준으로 환산되어, 슬롯에 무기가 없어도 픽업한 탄약이 누적되도록 한다.

본 시스템은 아래 경험을 보장한다.

- 같은 탄약 픽업이 무기 효율에 따라 다른 발사 횟수로 환산되는 경험
- 보조무기 슬롯이 비어 있어도 보조 탄약 픽업이 누적되는 경험
- 무기를 교체해도 모은 탄약량은 유지되며, 표시 발수만 새 무기 기준으로 갱신되는 경험
- 탄창은 무기 교체로 자동 충전되지 않고 재장전을 통해서만 채워지는 경험
- 탄창 용량이 무기 종류와 무관하게 일관된 디자인 기준에서 출발하는 경험

---

## 2. 탄약 분류와 슬롯 정책

### 2.1 탄약 분류

플레이어가 보유하는 탄약은 슬롯 단위로 두 종으로 분류한다.

| 탄약 분류 | 사용 슬롯 | 사용 무기 |
| --- | --- | --- |
| 주무기 탄약 | 주무기 슬롯 | 돌격소총, 볼트액션, 유탄발사기 |
| 보조무기 탄약 | 보조무기 슬롯 | 권총류 |

- 무기 종류별로 탄약을 별도 분류하지 않는다. 같은 슬롯에 들어가는 무기는 같은 탄약 풀을 공유한다.
- 무기 교체 시 같은 슬롯이라면 탄약은 그대로 유지된다.

### 2.2 슬롯 단위 정본

탄약은 어떤 무기를 장착했는지와 무관하게 슬롯 단위로 누적된다.

- 보조무기 슬롯이 비어 있어도 보조 탄약을 픽업할 수 있다.
- 슬롯에 무기를 장착하면 그 시점부터 보유 탄약이 새 무기 기준 표시 발수로 환산되어 보인다.
- 슬롯에서 무기를 해제해도 슬롯 탄약은 남아 있다.

---

## 3. 효율 시스템

### 3.1 효율 단가

같은 탄약 자원 1단위가 무기별로 몇 발에 해당하는지를 무기 데이터의 **효율 단가**로 정의한다.

- 효율 단가가 낮을수록 같은 자원으로 더 많이 발사 가능 (효율 좋은 무기)
- 효율 단가가 높을수록 같은 자원으로 적게 발사 가능 (효율 나쁜 무기)
- 디자인 baseline은 `1.0`으로, 30발 탄창의 일반 돌격소총이 이 값을 갖는다고 가정한다.

### 3.2 무기별 효율 예시

| 무기 분류 | 효율 단가 | 디자인 의도 |
| --- | --- | --- |
| 일반 돌격소총 | 1.0 | baseline |
| 고연사 돌격소총 | 1.2 | 빠른 연사로 자원 소모 ↑ |
| 정밀 사격 돌격소총 | 0.8 | 한 발의 가치 ↑, 자원 효율 ↑ |
| 볼트액션 | 약 3.3 | 적은 표시 발수, 발당 자원 소모 ↑ |
| 유탄발사기 | 약 16.7 | 매우 적은 표시 발수, 발당 자원 소모 매우 ↑ |
| 일반 권총 | 약 8.3 | 보조무기 baseline (12발 탄창 기준) |

- 효율 단가는 무기 데이터에서 정의하며, 무기 패시브나 모드에 의해 변동될 수 있다.

### 3.3 보유 한도 비율

탄창 용량 대비 보유 가능한 예비 탄약 비율을 무기 데이터의 **보유 한도 비율**로 정의한다.

- 비율이 5라면 탄창 5개분의 예비 탄약을 보유할 수 있다.
- 무기마다 다른 비율을 가질 수 있으며, 캐릭터 perk로 보정될 수 있다.

## 4. 탄창 용량과 보유 한도

### 4.1 탄창 baseline

모든 무기의 탄창 자원 baseline은 동일하다. 표시 발수는 효율 단가에 따라 결정된다.

- 모든 무기의 탄창은 자원 baseline `100` 단위를 가진다 (내부 기준값).
- 효율 단가 `1.0` 무기의 탄창 표시 발수 = `100` ÷ `1.0` = **100발**
- 효율 단가 `3.33` 무기의 탄창 표시 발수 = `100` ÷ `3.33` ≈ **30발**
- 효율 단가 `16.7` 무기의 탄창 표시 발수 = `100` ÷ `16.7` ≈ **6발**

이 구조 덕분에 디자이너는 무기 데이터에 별도의 탄창 발수 값을 입력하지 않고, 효율 단가만 조정해 표시 발수를 결정한다.

### 4.2 무기별 탄창·예비 한도 예시

`보유 한도 비율 = 5` 기준으로 정리한다.

| 무기 분류 | 효율 단가 | 표시 탄창 | 표시 예비 한도 |
| --- | --- | --- | --- |
| 일반 돌격소총 | 1.0 | 100발 | 500발 |
| 정밀 돌격소총 | 0.8 | 125발 | 625발 |
| 볼트액션 | 3.33 | 30발 | 150발 |
| 유탄발사기 | 16.7 | 6발 | 30발 |
| 일반 권총 | 8.33 | 12발 | 60발 |

- 표시 예비 한도는 효율 단가가 낮을수록 자연스럽게 커진다 (자원 baseline은 무기 무관하게 동일하므로).
- 캐릭터 perk가 보유 한도 비율을 늘리면 모든 무기 슬롯의 표시 예비 한도가 비례 증가한다.

## 5. 재장전 룰

### 5.1 기본 흐름

`[시스템 기획] 전투_사격_판정.md`의 재장전 흐름을 기반으로, 본 시스템은 자원 이동 단위만 정의한다.

- 재장전은 **표시 발수 단위로 부족분만큼 슬롯 자원에서 탄창 자원으로 이동**한다.
- 부족 표시 발수는 `(표시 탄창 한도) − (현재 표시 탄창)` 으로 계산한다.
- 보유 표시 발수는 `(현재 슬롯 자원)` 을 효율 단가로 나누어 산출한다.
- 실제 이동 발수는 `min(부족 표시 발수, 보유 표시 발수)` 이다.

### 5.2 자원 이동 규칙

- 한 발에 해당하는 자원량은 **효율 단가** 와 같다.
- 재장전 이동 발수가 `N`이면 슬롯에서 `N × 효율 단가` 만큼 자원이 빠지고 탄창에 같은 양이 들어간다.
- 표시 발수가 0이 될 때까지만 이동하고, 자원이 부분 단위로 남아있어도 추가 발사에는 사용되지 않는다.

### 5.3 특수 케이스

- 볼트액션은 한 발씩 재장전되므로, 이동 단위는 `1 × 효율 단가` 이다.
- 유탄발사기처럼 효율 단가가 큰 무기에서 보유 자원이 1발에 못 미치면 재장전이 시작되지 않는다.
- 탄창이 가득 찬 상태에서 재장전 입력은 무시한다.

## 6. 픽업 룰

### 6.1 픽업 자원 단위

탄약 픽업 액터는 **자원량(raw)** 단위로 적힌 값을 가진다.

- 픽업이 `100` 자원량을 가지면, 효율 단가 `1.0` 무기로는 100발, `3.33` 무기로는 30발 분량이다.
- 픽업 종류 (소형/중형/대형)는 자원량으로 차등화한다.

| 픽업 분류 | 자원량 | 일반 돌격소총 환산 | 볼트액션 환산 |
| --- | --- | --- | --- |
| 소형 | 50 | 50발 | 15발 |
| 중형 | 100 | 100발 | 30발 |
| 대형 | 250 | 250발 | 75발 |

### 6.2 적립 규칙

- 픽업은 슬롯 자원에 직접 가산된다 (효율 단가 적용 없음).
- 슬롯에 무기가 장착되어 있지 않아도 픽업이 가능하며, 자원은 슬롯에 누적된다.
- 자원이 슬롯의 raw 보유 한도에 도달하면 초과분은 픽업되지 않고 픽업 액터에 남는다.
- 픽업 액터는 모든 자원이 흡수되면 사라지고, 일부만 흡수되면 잔여 자원량을 표시한 채로 남는다.

### 6.3 슬롯 한도와 잔여 처리

- 보유 한도가 있는 상태에서 픽업이 한도를 초과하면, 픽업 액터는 잔여량을 유지한 상태로 월드에 남는다.
- 무기 미장착 슬롯에 대한 보유 한도 정책은 별도 운용한다 (현 단계에서는 미장착이어도 적립 가능, 한도 클램프는 실제 무기 장착 시 적용 예정).

## 7. 무기 교체 시 자원 처리

| 항목 | 처리 |
| --- | --- |
| 슬롯 탄창 자원량(raw) | 유지 (자동 재충전 없음) |
| 슬롯 예비 자원량(raw) | 유지 |
| 효율 단가 / 보유 한도 비율 | 새 무기 데이터 값으로 갱신 |
| 표시 탄창 한도 | 새 효율 단가 기준으로 재계산 |
| 표시 예비 한도 | 새 효율 단가와 보유 한도 비율로 재계산 |
| 현재 표시 탄창 | 슬롯 탄창 자원을 새 효율 단가로 나누어 표시 |
| 현재 표시 예비 | 슬롯 예비 자원을 새 효율 단가로 나누어 표시 |

- 무기 교체 시 슬롯의 raw 자원은 어떤 경우에도 자동 충전되지 않는다. 효율 단가와 보유 한도 비율만 갱신된다.
- 교체 직후 표시값이 무기 효율에 따라 달라 보일 수 있다 (예: 효율 좋은 무기로 교체 시 표시 예비 발수가 늘어 보임).
- 슬롯이 비어 있던 상태에서 무기를 처음 장착해도 탄창은 비어 있는 상태로 시작한다. 첫 사격 전 재장전이 필요하다.
- 탄창에 자원을 채우는 유일한 경로는 재장전이며, 재장전은 슬롯의 예비 자원을 소모해 탄창을 채운다.

## 8. HUD 표시 규칙

### 8.1 표시 단위

HUD에서 보여 주는 모든 탄약 수치는 **표시 발수** 기준이다. 자원량(raw)은 플레이어에게 노출하지 않는다.

| 표시 항목 | 계산 |
| --- | --- |
| 현재 탄창 발수 | `floor(현재 탄창 자원 ÷ 효율 단가)` |
| 탄창 한도 | `floor(자원 baseline ÷ 효율 단가)` |
| 현재 예비 발수 | `floor(현재 슬롯 자원 ÷ 효율 단가)` |
| 예비 한도 | `floor(자원 baseline × 보유 한도 비율 ÷ 효율 단가)` |

### 8.2 표시 정책

- 활성 슬롯의 탄창과 예비 발수를 강조 표시하고, 비활성 슬롯의 예비 발수는 부가 표시로 둔다.
- 효율 단가가 갱신되면 (무기 교체, 패시브 변동) 표시 발수가 즉시 재계산되어 갱신된다.
- 보조무기 슬롯이 비어 있어도 보조 예비 자원은 누적될 수 있으나, 표시 발수는 효율 단가가 결정되지 않은 상태이므로 자원량 그 자체가 아닌 **자원량(raw)** 또는 "무기 미장착" 표시로 대체한다.
